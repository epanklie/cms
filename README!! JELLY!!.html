Pengenalan

  1. pertama lu buka file autoload.php, ada di application/config/autoload.php. kalo lu males cari bisa pake Ctrl+P trus ketik aja nama file nya
    liat di situ ada $autoload['libraries'] = array('database', 'session', 'pagination');
    nah itu fungsinya buat ngeload otomatis library dari database, session, pagination
    kalo itu ga di pake, misalkan lu butuh fungsi library dari database, lu bakal manggil itu di setiap controller atau model, nahh itu kan ribet, mendingan di autoload
    yang $autoload['helper'] = array('url', 'form', 'file'); fungsi nya juga sama, dia buat ngeload otomatis, cuman kalo dia ngeload yg namanya helper, bukan library

  2. trus lu liat file config.php, di situ lu liat $config['base_url'] = ''; itu sengaja gw ga isi biarkan dia default, biar dia ngelink nya otomatis sesuai nama folder sama nama localhost lu

  3. lu liat lagi file database.php, di situ lu isi username, password, sama nama database jadi phpmyadmin lu. defaultnya lepi lu sihh biasanya usernamenya root passwordnya kosong

  4. buka file routes.php, disitu ada default controller yang fungsi nya ngecek controller pertama yang mau di jalanin. kalo lu liat:
    $route['register'] = 'login/register';
    $route['logout'] = 'login/logout';
    $route['search'] = 'page/search';
     itu buat manipulasi link/url nya, jadi biasanya kan kalo link itu nama controller baru nama function, nahh kalo pake itu, jadi kita bisa ganti, misalkan controllernya page trus functionnya search, tapi lu mau nanti linknya itu yg muncul cuman nama functionnya doang, nahh, lu bisa manipulasi dengan cara $route['search'] = 'page/search';



Controller

  1. disitu ada 3 controller, dashboard, login, page, controller dashboard itu buat control backendnya, kalo controller login, itu buat control login, register sama logout. kalo page itu buat control frontendnya.

  2. Controller Page.php
    disitu ada beberapa function, nahh function2 itu yg akan jadi link yang kaya gw bilang tadi, contoh: http://localhost/ci_jelly/nama_controller/nama_function
    disitu ada yang namanya function __construct(), itu buat jalanin yg pertama, dan juga di jalanin buat ke semua function lain yang ada di dalem controller tsb. Nahh di dalem function itu, isi nya ada $this->load->model('page_model'); itu buat ngeload ke semua function yg ada di controller tsb, jadi setiap function yang manggil $this->page_model->nama_function_model itu ga perlu ngeload lagi satu persatu di setiap function.
    di situ ada $data['blabla'] trus ada $this->session nah itu nanti gw jelasin pas hari sabtu aja kalo lu ketemu gw, daripada gw ngetik disini, selain ribet, belom tentu juga penjelasan gw jelas :v

  3. Controller Login.php
    nanti gw jelasin pas gw ketemu lu hari sabtu.

  4. Controller Page.php
    nanti gw jelasin pas gw ketemu lu hari sabtu.
